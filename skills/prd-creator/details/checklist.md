# PRD Creation  -  Checklist

## Definition of Done

### Problem and Scope
- [ ] Problem statement is clear and specific
- [ ] Goals are measurable and aligned to the problem
- [ ] Non-goals are explicit

### Users and Journey
- [ ] Primary user is defined
- [ ] User journey is documented
- [ ] Edge cases are noted
- [ ] User stories are included
- [ ] Workflows are documented

### Requirements
- [ ] Functional requirements are testable
- [ ] Non-functional requirements are stated
- [ ] Priorities are expressed as Must/Should/Could

### Architecture and Testing
- [ ] Architecture summary is included
- [ ] Test strategy is defined
- [ ] Production requirements are documented

### UX and Content
- [ ] Screens and states are listed
- [ ] Accessibility and localization are addressed
- [ ] Links to design assets are included (if available)

### Data and Integrations
- [ ] Required data inputs/outputs are documented
- [ ] API or integration dependencies are listed

### Metrics and Rollout
- [ ] Success metrics and measurement plan are defined
- [ ] Rollout plan is stated
- [ ] Risks and mitigations are captured
- [ ] PRFAQ is present (press release + FAQ)
- [ ] PRD path and TODOs are included

### Review
- [ ] Stakeholder review completed
- [ ] Open questions have owners
